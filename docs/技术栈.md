# 云原生博客与知识库系统 - 技术栈说明

本文档描述项目所采用的技术选型及版本规划，便于开发与答辩时统一口径。

---

## 一、总体架构

- **架构风格**：微服务
- **通信方式**：RESTful API（HTTP/JSON）+ 可选异步消息
- **部署与运维**：Docker 容器化 + Kubernetes 编排

---

## 二、后端技术栈

### 2.1 基础框架

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **Java** | 开发语言 | JDK 17 或 21（LTS） |
| **Spring Boot** | 应用框架 | 3.x（与 JDK 17+ 匹配） |
| **Spring Cloud** | 微服务治理 | 与 Spring Boot 版本对应的 Alibaba 或 2023.x 系列 |
| **Maven** | 构建与依赖管理 | 3.8+ |

### 2.2 数据访问

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **MyBatis Plus** | ORM / 持久层 | 基于 MyBatis，简化 CRUD、分页、条件构造 |
| **MySQL** | 主数据存储 | 8.x，存储用户、文章、分类、标签、评论等 |
| **Druid** | 数据源与监控 | 连接池、SQL 监控（可选） |

### 2.3 缓存与会话

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **Redis** | 缓存 / 会话 / 计数 | 缓存热点文章、会话、点赞/阅读量等 |

### 2.4 安全与认证

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **Spring Security** | 认证与权限 | 登录、鉴权、接口保护 |
| **JWT** | 无状态令牌 | 前后端分离下的 Token 方案（如 jjwt） |

### 2.5 其他后端组件

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **Lombok** | 简化实体与样板代码 | 减少 getter/setter、构造器等 |
| **Hutool** | 工具类 | 日期、字符串、加密、HTTP 等（可选） |
| **Knife4j / Swagger** | API 文档 | 自动生成 OpenAPI 文档与在线调试 |
| **Validation** | 参数校验 | Bean Validation（@NotNull、@Size 等） |

## 四、云原生与基础设施

### 4.1 容器与编排

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **Docker** | 容器化 | 每个微服务构建为独立镜像 |
| **Kubernetes (K8s)** | 编排与部署 | 部署、服务发现、负载均衡、弹性伸缩、滚动更新 |

### 4.2 微服务治理（满足“至少 3 种云原生组件”）

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **Nacos** | 服务注册与发现、配置中心 | 阿里开源，可同时做注册中心与配置管理 |
| **Spring Cloud Gateway** | API 网关 | 路由、鉴权、限流、统一入口 |
| **OpenFeign** | 服务间 HTTP 调用 | 声明式调用其他微服务 |
| **Sentinel 或 Resilience4j** | 熔断与限流 | 容错与高可用 |

### 4.3 存储与中间件（按需选型）

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **Elasticsearch** | 全文搜索 | 文章、知识库检索（任务书中的“搜索服务”） |
| **MinIO** | 对象存储 | 图片、附件上传（任务书中的“文件存储服务”） |
| **RabbitMQ 或 Kafka** | 异步消息 | 评论通知、文章同步到 ES 等（可选） |

### 4.4 可观测性

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **SLF4J + Logback** | 日志 | 统一日志格式与输出 |
| **Spring Boot Actuator** | 健康检查与指标 | K8s 探针、Prometheus 拉取（可选） |
| **SkyWalking 或 Zipkin** | 链路追踪 | 分布式调用链（可选，用于答辩亮点） |

### 4.5 持续集成 / 持续部署（CI/CD）

| 技术 | 用途 | 说明/版本建议 |
|------|------|----------------|
| **GitHub Actions / GitLab CI / Jenkins** | 流水线 | 构建、测试、构建镜像、推送到仓库 |
| **Harbor 或 Docker Registry** | 镜像仓库 | 存放 Docker 镜像（可选，本地可用 Docker Hub） |

## 六、与任务书的对应关系

- **微服务数量 ≥ 3**：用户认证服务、博客内容服务、评论服务；可扩展：文件服务、搜索服务。
- **容器化 + K8s**：所有服务 Docker 化，由 Kubernetes 统一编排。
- **云原生组件 ≥ 3**：如 Nacos（注册+配置）、Gateway（网关）、Sentinel（熔断限流）、可观测（日志/指标/链路）中任选 3 类即可。
- **CI/CD**：至少包含构建、测试、镜像构建与（可选）部署到 K8s 的流水线。
